# Release 1.1.0 — Post-launch hardening

**Branch:** `release/1.1.0` (from `dev`)  
**Target:** merge to `dev` → `main` after QA  
**Baseline:** v1.0.0 is live in production (55 migrations applied, Vercel prod deployed)

Release **1.1.0** is the first maintenance minor after go-live. Scope is **gap closure and production hardening** called out in the v1 launch PR and implementation plan — not new Phase 2/3 product surface.

## Merge & deploy sequence

1. Open PR: `release/1.1.0` → `dev`
2. CI green + manual test plan above
3. Merge to `dev`, then PR `dev` → `main`
4. Apply migration **0056** on PROD Supabase
5. Set `RESEND_WEBHOOK_SECRET` in Vercel Production (if not already)
6. Deploy Vercel production from `main`
7. Post-deploy smoke (5 min)

## Risks

| Risk                                                   | Mitigation                                                              |
| ------------------------------------------------------ | ----------------------------------------------------------------------- |
| Prepay invoice duplicates                              | Unique index on `(tenant_id, quote_id)` where quote_id not null         |
| Resend webhook breaks campaign metrics if secret wrong | Log verification failures; document secret rotation in Resend dashboard |
| PROD already has data                                  | Migration is additive only (nullable columns + index)                   |
