# Release 1.2.0 — Interaction latency (Phase 1)

**Branch:** `release/1.2.0` (from `dev`)  
**Plan:** [`docs/performance/interaction-latency-plan.md`](../performance/interaction-latency-plan.md)

---

## Summary

First performance release targeting **perceived click latency** and **redundant server work**:

- Route-level **`loading.tsx`** skeletons for tenant + customer portals (instant click feedback)
- **`React.cache()`** on `getAuthContext()` — dedupe auth fetch in layout + pages
- **Customer layout** pending-quote badge uses **count query** instead of full quote list
- **Removed `revalidatePath(..., 'layout')`** from server actions (19 call sites) — stops full shell invalidation on every form submit
- **Quotes Kanban optimistic updates** — cards move immediately; no `router.refresh()` on drag
